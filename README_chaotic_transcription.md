# 混乱学生讨论转录生成器

## 问题诊断

之前的脚本存在以下问题：
1. ❌ **重复率过高**：大量内容重复，缺乏多样性
2. ❌ **模式化严重**：总是以"不对"、"那个"开头
3. ❌ **缺少随机性**：相同主题总是生成相似内容
4. ❌ **不够自然**：缺乏真实学生讨论的自然流畅度

## 优化方案

### 1. 增加Few-shot示例多样性
- **之前**：只有4个固定示例
- **现在**：每门课程7个不同示例，涵盖各种混乱模式

### 2. 改进Prompt设计
- 添加随机风格提示（窃窃私语、抢答、自言自语等）
- 添加随机混乱模式（自我否定、被打断、思维跳跃等）
- 明确语气要求（犹豫、疑惑、试探、自言自语）

### 3. 提高随机性参数
```python
temperature: 1.5  # 从1.3提升到1.5
top_p: 0.9         # 新增，控制多样性
```

### 4. 优化内容处理
- 自动清理多余符号
- 智能截取保持句子完整
- 确保正确标点结尾

### 5. 保留API调用（保持准确性）
- `get_lesson_type`：使用API准确判断课程类型
- `get_lesson_topic`：使用API准确提取课程主题

## 使用方法

### 基本使用
```bash
python3 generate_chaotic_transcriptions.py
```

### 测试生成结果
```bash
python3 test_chaotic_transcription.py
```

测试脚本会分析：

- 重复率（目标：<20%）
- 模式分析（"不对"、"那个"等使用频率）

## 参数调整建议

### 如果仍然重复
在 `generate_chaotic_transcription` 函数中调整：
```python
temperature: 1.8  # 进一步提高随机性
top_p: 0.95        # 增加多样性
```

### 如果内容过于混乱
```python
temperature: 1.2  # 降低随机性
top_p: 0.85        # 减少多样性
```

### 如果长度不符合要求
在prompt中调整：
```python
# 20-25个汉字
# 改为
# 15-20个汉字（更短）
# 或
# 25-30个汉字（更长）
```

## 混乱风格说明

### 风格1：窃窃私语型
特点：小声说话，断断续续
示例："呃斜率是正切值对然后那个..."

### 风格2：抢答纠正型
特点：抢话，互相纠正，观点冲突
示例："不对那个是Δy比Δx不等等是变化量..."

### 风格3：自言自语型
特点：自我否定，来回纠结，反复修改
示例："斜率是k不对吧它是垂直等等..."

### 风格4：思维跳跃型
特点：突然想起相关知识点，插话说
示例："那个公式不对等一下想到另一个..."

### 风格5：打断型
特点：说一半被打断，转而说别的
示例："直线斜率一二一二...哦不对坐标原点..."

## 常见问题

### Q1: 为什么还是有重复？
**A:** 可以增加temperature到1.8，或手动为相同主题添加更多示例。

### Q2: 内容太混乱看不懂
**A:** 降低temperature到1.2，调整prompt要求更连贯。

### Q3: API调用太慢
**A:** API调用是必要的，用于准确判断课程类型和主题。可以：
   - 一次只处理几门课程测试
   - 增加timeout时间（目前30秒）
   - 确保网络连接稳定

### Q4: 生成的字数不对
**A:** 检查max_tokens设置，或调整prompt中的字数要求。
