# eduPDF 改进方案需求文档

## 1. 项目概述

### 1.1 项目背景
- **项目名称**: eduPDF - 教育场景多模态课堂行为识别系统
- **当前架构**: BERT + ResNet50 + LSTM + PDF动态融合框架
- **核心问题**: 
  - 28.5%样本存在空文本（1798/6318）
  - 极端类别不平衡（1577:38，41:1）
  - 小样本类别（技术操作仅38条）
  - LSTM无法处理长视频序列
  - PDF全息权重硬编码，不可学习

### 1.2 改进目标
- **准确率提升**: 75% → 88-93%（提升13-18个百分点）
- **空文本样本**: 准确率提升25-30%
- **小样本类别**: 准确率提升15-20%
- **推理速度**: 提升5倍（100ms → 20ms）
- **可解释性**: 提供五步过程监督评分

### 1.3 数据现状
```
总样本: 6318 (原3787，增长67%)
类别分布:
- 教师讲授: 1577条
- 学生讨论: 807条
- 教师板书: 623条
- 教师提问: 584条
- 学生发言: 423条
- 课堂沉寂: 398条
- 教师反馈: 161条
- 教师巡视: 161条
- 教师指令: 146条
- 技术操作: 38条 ⚠️

模态完整性:
- video_description: 6313/6318 (99.9%) ✅
- text: 4520/6318 (71.5%)
- 空文本: 1798/6318 (28.5%) ⚠️
```

## 2. 核心需求

### 2.1 功能需求

#### FR-1: 三模态融合
**优先级**: P0（必须）
**描述**: 系统性利用video_description作为第三模态
**验收标准**:
- [ ] 支持text、video_description、video三路输入
- [ ] 空文本时自动使用video_description作为回退
- [ ] 非空文本时支持双段拼接输入
- [ ] 三模态融合准确率提升5-8%

#### FR-2: LLM数据增强
**优先级**: P0（必须）
**描述**: 使用LLM生成小样本类别的训练数据
**验收标准**:
- [ ] 技术操作: 38 → 200条
- [ ] 教师反馈: 161 → 400条
- [ ] 生成数据质量人工审核通过率 > 80%
- [ ] 小样本类别准确率提升15-20%

#### FR-3: Mamba序列编码
**优先级**: P0（必须）
**描述**: 用Mamba替换LSTM处理视频时序
**验收标准**:
- [ ] 支持100+帧视频输入（原16帧）
- [ ] 推理速度提升5倍
- [ ] 内存占用降低60%
- [ ] 准确率提升3-5%

#### FR-4: Sparse MoE ConfidNet
**优先级**: P0（必须）
**描述**: 用稀疏MoE替换20个独立置信度网络
**验收标准**:
- [ ] 8个共享专家（2共享+2模态+4场景）
- [ ] Top-2稀疏激活
- [ ] 模态感知路由（text=""自动路由到视频专家）
- [ ] 参数量降低50%，准确率提升8-12%

#### FR-5: 层次化MoE融合层
**优先级**: P0（必须）
**描述**: 将PDF硬编码全息权重升级为可学习的MoE路由
**验收标准**:
- [ ] Layer1: 模态路由MoE（3个专家）
- [ ] Layer2: 类别路由MoE（10个专家）
- [ ] 保留PDF核心思想（动态调整权重）
- [ ] 准确率提升8-12%

#### FR-6: PRM过程监督
**优先级**: P1（重要）
**描述**: 引入过程奖励模型监督融合过程
**验收标准**:
- [ ] 五步过程监督（文本质量、视频质量、对齐度、路由、融合）
- [ ] LLM生成监督信号（GPT-4o-mini）
- [ ] 输出每步评分（可解释性）
- [ ] 准确率提升12-18%

### 2.2 非功能需求

#### NFR-1: 性能要求
- 推理速度: ≤ 20ms/样本
- 训练速度: ≤ 10小时/epoch（单卡V100）
- 内存占用: ≤ 16GB（训练）, ≤ 4GB（推理）

#### NFR-2: 可扩展性
- 支持新增类别（无需重新训练整个模型）
- 支持新增模态（如音频）
- 支持不同视频长度（16-100帧）

#### NFR-3: 可解释性
- 输出融合权重（w_text, w_video）
- 输出MoE路由决策（激活了哪些专家）
- 输出PRM五步评分
- 支持可视化分析

#### NFR-4: 鲁棒性
- 处理缺失模态（text=""或video损坏）
- 处理类别不平衡
- 处理小样本类别

## 3. 用户故事

### US-1: 作为研究者，我希望模型能处理空文本样本
**场景**: 28.5%的样本text=""，导致BERT编码失败
**期望**: 自动使用video_description作为文本输入
**验收**: 空文本样本准确率从40%提升到65-70%

### US-2: 作为研究者，我希望模型能识别小样本类别
**场景**: 技术操作仅38条，模型无法学习
**期望**: 使用LLM生成200条高质量样本
**验收**: 技术操作准确率从30%提升到50-60%

### US-3: 作为研究者，我希望模型能处理长视频
**场景**: 教师板书需要30秒视频，但LSTM只能处理16帧
**期望**: Mamba支持100帧输入
**验收**: 教师板书准确率提升8-12%

### US-4: 作为研究者，我希望融合权重可学习
**场景**: PDF全息权重是固定公式，无法适应不同场景
**期望**: 用MoE路由替代硬编码公式
**验收**: 融合权重根据样本动态调整，准确率提升8-12%

### US-5: 作为研究者，我希望知道模型为什么这样预测
**场景**: 模型预测错误，但不知道哪步出问题
**期望**: PRM输出五步评分，定位问题
**验收**: 输出文本质量、视频质量、对齐度、路由、融合五个评分

## 10. 附录

### 10.1 术语表
- **PDF框架**: Probabilistic Distribution Fusion，概率分布融合
- **全息权重**: Holographic Weights，通过对数比值计算的互补权重
- **MoE**: Mixture-of-Experts，混合专家模型
- **PRM**: Process Reward Model，过程奖励模型
- **Mamba**: 基于状态空间模型的高效序列建模方法

### 10.2 参考文献
1. Mamba: Linear-Time Sequence Modeling with Selective State Spaces (2023)
2. Sparse Mixture of Experts (GShard, 2020)
3. Process Reward Models (OpenAI o1, 2024)
4. PDF: Probabilistic Distribution Fusion (原论文)
